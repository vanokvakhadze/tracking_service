# ლოკაცია + გეოფენსი

ეს guide-ი ხსნის TrackPro-ის გეოფენსის სისტემას — როგორ ვაცნობოთ აპლიკაციას სად მუშაობს თანამშრომელი + როგორ ვართ დარწმუნებული რომ ცვლა ფაქტობრივად ფიქსდება.

## Two-zone pattern (ცვლის ცენტრი + სამუშაო ზონა)

თითო ლოკაცია **ორ წრეს** იღებს რუკაზე:

| ზონა | რა აკეთებს | რადიუსი | ფერი |
|------|----------|--------|------|
| **Trigger** (ცვლის ცენტრი) | აქ შემოსვლა → SHIFT_START. გასვლა → SHIFT_END | 50–1500 მ | KAYA blue |
| **Boundary** (სამუშაო ზონა) | აქ შემოსვლა → "ახლოს ხართ" push. გასვლა → alert ადმინს | 100–5000 მ | Amber |

წესი: **Trigger ≤ Boundary**. UI არ მოგცემთ მართალე საპირისპიროდ.

## რად გვჭირდება ორი ზონა?

ცხოვრებაში GPS arrival/departure არ არის "სუფთა":

- თანამშრომელი წავიდა ლანჩზე → ფიზიკურად 50მ გავიდა Trigger-ის გარეთ → ცვლა არ უნდა დასრულდეს
- მიდის ოფისთან მანქანით → Boundary-ში გადადის, მაგრამ ჯერ კიდევ მანქანაში → ცვლა არ უნდა იწყებოდეს

ამიტომ:

- **Trigger** — "მიდის work-ში" სიგნალია
- **Boundary** — "ცოტა ახლოს მოვედი" სიგნალია

ცვლა მხოლოდ Trigger-ის შემოსვლა-გასვლის გადასწავლაა.

## Hysteresis (debouncing)

GPS drift-ის წინააღმდეგ:

- **Entry confirmation**: უნდა იყო Trigger-ში **30 წამი** — შემდეგ SHIFT_START
- **Exit confirmation**: უნდა იყო გარეთ **60 წამი** — შემდეგ SHIFT_END

ეს ნიშნავს, რომ:

- ნუ ცდახ GPS-ის chairs-ში 5 წამიანი "blip" გასვლა — ფიქსირდება
- მცირე ჩუხჩუხი ცვლის სანდოობის ფარგლებში დარჩება

## რადიუსის ბესტ პრექტისი

| ლოკაციის ტიპი | Trigger | Boundary |
|-------------|---------|----------|
| ოფისი | 100 მ | 200 მ |
| საწყობი | 150 მ | 300 მ |
| სამშენებლო მოედანი | 250 მ | 500 მ |
| დიდი კომპლექსი | 300 მ | 600 მ |
| **Default** | **100 მ** | **200 მ** |

**წესი:** ნუ აყენებ Trigger < 50 მ — GPS accuracy მაგრე ცუდი ხდება, false trigger-ი გარანტირებულია.

## Accuracy buffer

GPS-ის accuracy მუდმივი არ არის. TrackPro-ის ფორმულა:

```
effectiveRadius = trigger.radius + (gps.accuracy × 0.5)
```

ანუ თუ Trigger = 100მ, GPS accuracy = 15მ → ნამდვილი ფარგლები 107.5მ-მდე. ეს ფარგი user-ისთვის არ ჩანს, მაგრამ ცვლის ფიქსაცია უფრო სანდო ხდება.

## ლოკაციის შექმნა — admin flow

1. **ლოკაცია** → **+ ახალი**
2. **Map mode** ან **Address mode**
3. რუკაზე — pin გადათრევით ან ცენტრის ცვლა
4. სახელი, კატეგორია, ორი slider Trigger + Boundary-სთვის
5. **შენახვა** → `status='active'`, push-ი ეგზავნება ყველა employee-ს

## პროვიზიული ლოკაცია — employee flow

თუ თანამშრომელი მიდის ისეთ ადგილზე, რომელიც ჯერ არ არის registered:

1. Mobile app → **მონიშნე ლოკაცია**
2. ფოტო (selfie ან building) — GPS coordinates ფოტოს მეტადატაში ჩაიწერება
3. შენიშვნა (optional)
4. **გადააგზავნე დასამტკიცებლად**

Admin იღებს push notification → **მოლოდინში** inbox → თვალს ადევნებს → **დაამტკიცე** ან **უარყავი**.

დამტკიცებული provisional location ხდება სრულფასოვანი location-ი + employee-ის ცვლა ჩაიწერა.

## სამუშაო ზონის რედაქტირება

ლოკაცია → **სამუშაო ზონა** ღილაკი → ცვალე Trigger ან Boundary radius — გადარჩება. **შენახვა** ხდება მაშინვე და ცოცხალი employee-ის ცვლები ცვლის ფარგლებს real-time-ში.

## Battery & ბატარეის ეკონომია

TrackPro motion-based state machine-ით მუშაობს:

- **Stationary** (ჩაცული) → GPS გათიშულია, აპლიკაცია "სძინავს"
- **Moving** (მოძრაობს) → GPS ჩართულია, ping-ი 30 წამიანი

ეს იძლევა ~70% battery savings ჩვეულებრივ continuous tracking-თან შედარებით.
